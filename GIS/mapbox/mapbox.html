# Mapbox gl 入门 一

# 说明

本文适合初次尝试使用Mapbox或GIS地图库的前端开发者，如果你是一名准备入行GIS或者在校GIS专业学生，也可以通过此文章了解Mapbox。

# 简介

mapbox gl是一款优秀的开源GIS开源库，区别于Cesium，它具有三维的表现能力，又不能算真正的三维地图，介于二维与三维之间，它与我们所熟知的百度高德地图非常相似，由于它优秀的加载速度和可扩展性，以及创新性的矢量瓦片(一种数据格式)，令它成为非常流行的开源GIS框架。

# 为什么选择Mapbox

  1. 如果你只是想在自己的项目中使用地图简单的展示一些地理位置信息，并实现简单的交互效果，你可以选择它。但如果你需要导航等功能，我更建议使用百度高德这些商业化的地图框架。
  2. 如果项目要求使用开源库，并且自己需要展示的不是特大场景的数据（全国或全球范围即特大场景，或不涉及以球的形式展示），你可以选择Mapbox

# 代码逻辑

数据源（Source）--->图层（Layer)--->地图（Map)

在实际去写代码前，还需了解地图加载逻辑，这里所说的逻辑指的是代码书写逻辑，而非真正的渲染逻辑。简单来讲，假如你手里有一些拥有全国污染情况的数据，你将这些数据托管在数据源（Source）中，以不同类型的污染源分成多个图层（Layer），设置图层的样式，最后加载至地图（Map)上便完成了数据的展示。

# 初始化地图实例（配置)

# 安装

npm install --save mapbox-gl

# 引入

<link href='https://api.mapbox.com/mapbox-gl-js/v1.12.0/mapbox-gl.css' rel='stylesheet' />

import mapboxgl from 'mapbox-gl';

# 创建地图容器并初始化

<div id="map"></div>

mapBox.accessToken = 'xxx';
var map = new mapBox.Map(MapConfig);

官方初始化示例 (opens new window) 这样我们便完成了地图和图层的加载，如果想要完成的复杂的渲染形式，需要自己去花时间了解地图样式 (opens new window) 事实上，mapbox所有图层的加载与渲染都可以通过配置图层去完成，对于初学者来说这也是最复杂的一部分，大家可以参考官方示例实现更加复杂的功能

# 交互

在默认底图加载后，我们还需要做一些交互功能

# 加载图层（动态）

但凡涉及加载图层，我们都需要在地图配置中配置它的数据源和图层样式，虽然也可以通过代码的方式添加数据源和图层，但我个人并不推荐这么做，这样并不利于图层的管理 在很多加载逻辑中，我们不会直接展示某些图层，一般会通过按钮的方式进行加载或隐藏（移除）
这是对一个图层显隐性进行控制，也可以将它直接移除

map.setLayoutProperty('layerId', 'visibility', 'none');// 隐藏
map.setLayoutProperty('layerId', 'visibility', 'visible');// 显示
map.getLayer('layerId').remove()// 移除图层

有些数据是接口数据，其返回格式并不能直接加载，需要手动拼接成标准Geojson格式

const geoJsonData = {
       type: 'FeatureCollection',
       features: [],//将接口数据灌入这里，一定要注意单要素与多要素的数组嵌套
    };
map.getSource('layerId').setData(geoJsonData);

# 飞行

地图飞行事件，需要设置飞行坐标，以及缩放级别，倾角等

map.flyTo({
       center: [120,30],
       zoom: 13,
       pitch: 60,
     });

# 监听事件

地图监听事件可以满足很多交互应用场景，最常见的是点击某一要素，高亮或者传给前端组件一个状态值。

map.on('click', () => {
    //do...
});

地图上不光有点击，也会有移入、移出、地图加载等监听事件，具体需要查看Api文档
