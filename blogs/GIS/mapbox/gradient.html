mapbox面整体渐变 ​

简述 ​

当下的设计师们非常喜爱使用渐变色，从实际效果上也确实好看一些，不过地图上的渐变看似简单，实则有时让开发者十分头痛。 在mapbox里，最常见的应该是线面渐变，在已有的api里，线渐变可以轻松实现,但是面渐变，以及体（fill-extrusion）渐变就变得复杂很多，今天要讲解的是面图层整体渐变的一种实现方式。

整体逻辑 ​

首先要明确的是，当前实现的效果是整个图层的渐变，也就是不区分feature的个数。

  1. 新增两个自定义图层属性 fill-pattern-repeat,fill-bound
  2. 将瓦片的墨卡托范围传入shader
  3. 根据 u_repeat 判断是否执行即可

具体实现 ​

创建一个面图层,设置fill-pattern ，添加一个图片，默认效果下，图片会不断重复

js
// 添加图片
  map.loadImage("../debug/0.png", (err, image) => {
      if (err) throw err;
      map.addImage("bg0", image);
  });
  // addSource
  map.addSource("china", {
      type: "geojson",
      data,
  });
  // addLayer
  map.addLayer({
    id: "china",
    type: "fill",
    source: "china",
    layout: {},
    paint: {
        "fill-outline-color": "rgba(255, 255, 255, 1)",
        "fill-pattern": "bg0",
        "fill-opacity": 1,
    },
});


如果让图片根据一个属性判断是否让图片重复，那么就会实现想要的效果

添加 fill-pattern-repeat ​

  1. 在v8.json中找到fill相关属性，可以直接在fill-pattern下面添加一个fill-pattern-repeat,类型为boolean,默认为true,常量。
js
    "fill-pattern-repeat":{
      "type": "boolean",
      "default": true,
      "doc": "让材质不重复贴图, 而是按范围坐一次UV贴图",
      "transition": false,
      "sdk-support": {
        "basic functionality": {
          "js": "0.50.0",
          "android": "7.0.0",
          "ios": "4.7.0",
          "macos": "0.13.0"
        }
      },
      "expression": {
        "interpolated": false,
        "parameters": [
          "zoom"
        ]
      },
      "property-type": "data-constant"
    },

在fill_style_layer_properties.js 中设置相关属性

js
export type PaintProps = {|
    "fill-pattern-repeat": DataConstantProperty<boolean>,
|};

const paint: Properties<PaintProps> = new Properties({
    "fill-pattern-repeat": new DataConstantProperty(styleSpec["paint_fill"]["fill-pattern-repeat"]),
});

完成以上配置，mapbox就可以正确识别自定义属性

  1. 在fill_program.js中添加u_repeat
js
// 这一步不添加也行，这应该是ts中定义的类型
export type FillPatternUniformsType = {|
    'u_repeat': Uniform1f
|};
// 添加 uniform
const fillPatternUniforms = (context: Context): FillPatternUniformsType => ({
    'u_repeat': new Uniform1f(context),
});
// 额外添加一个参数，将layer作为参数传进来，同时在patternUniformValues也将layer作为参数传进去
const fillPatternUniformValues = (
    matrix: Float32Array,
    painter: Painter,
    tile: Tile,
    layer,
): UniformValues<FillPatternUniformsType> => extend(
    fillUniformValues(matrix),
    patternUniformValues(painter, tile, layer)
);

这个layer需要从调用fillPatternUniformValues的地方传进来，所以要在draw_fill.js中更改一下

js
//  drawFillTiles 函数中，有我们需要的layer,fillPatternUniformValues也是在这里调用的
// 大概102行左右，将layer传进去
if (!isOutline) {
    indexBuffer = bucket.indexBuffer;
    segments = bucket.segments;
    uniformValues = image ?
        fillPatternUniformValues(tileMatrix, painter, tile, layer) :
        fillUniformValues(tileMatrix);
}

虽然uniform有了，但它还未和fill-pattern-repeat真正连接起来，顶层传递进来的值，需要赋给u_repeat,打开pattern.js

js

function patternUniformValues(painter: Painter, tile: Tile, layer): UniformValues<PatternUniformsType> {
  // ....
}
// 在patternUniformValues return中添加u_repeat

    'u_repeat': layer.paint.get('fill-pattern-repeat'),
  };

至此，fill-pattern-repeat从顶层传进来，转换为uniform供shader使用

  1. 在fill_pattern.fragment.glsl中使用u_repeat
js
// ...

// ...

添加 fill-bound ​

原本的fill-pattern是通过点的瓦片内坐标获取图片的uv，这就会让图片按照一定的大小进行重复贴图，注意这里并不是说图片的uv完全等于瓦片的大小。如果我们想给整个图层贴一张不重复贴图，那么需要把整个图层的范围传进去，然后计算瓦片的范围，这样才能算出一个正确的UV

  1. 计算图层范围，这在mapbox中并没有现成的办法，如果数据源是服务，那么服务应提供图层的范围，这可能需要另外的接口调用。这里提供geojson图层范围的计算方法
js
            function getGeojsonBound(layerName) {
              // geojson source 需要在add时以json的形式传进去，如果是地址，_data并不能获取数据
                const data = map.getSource(layerName)._data;
                const bounds = [
                    [Infinity, Infinity],
                    [-Infinity, -Infinity],
                ];
                // 可以优化为for循环
                data.features.forEach((feature) => {
                    if (feature.geometry.type === "Point") {
                        bounds[0][0] = Math.min(
                            bounds[0][0],
                            feature.geometry.coordinates[0]
                        );
                        bounds[0][1] = Math.min(
                            bounds[0][1],
                            feature.geometry.coordinates[1]
                        );
                        bounds[1][0] = Math.max(
                            bounds[1][0],
                            feature.geometry.coordinates[0]
                        );
                        bounds[1][1] = Math.max(
                            bounds[1][1],
                            feature.geometry.coordinates[1]
                        );
                    } else if (feature.geometry.type === "LineString") {
                        feature.geometry.coordinates.forEach((coord) => {
                            bounds[0][0] = Math.min(bounds[0][0], coord[0]);
                            bounds[0][1] = Math.min(bounds[0][1], coord[1]);
                            bounds[1][0] = Math.max(bounds[1][0], coord[0]);
                            bounds[1][1] = Math.max(bounds[1][1], coord[1]);
                        });
                    } else if (feature.geometry.type === "Polygon") {
                        feature.geometry.coordinates.forEach((coords) => {
                            coords.forEach((coord) => {
                                bounds[0][0] = Math.min(bounds[0][0], coord[0]);
                                bounds[0][1] = Math.min(bounds[0][1], coord[1]);
                                bounds[1][0] = Math.max(bounds[1][0], coord[0]);
                                bounds[1][1] = Math.max(bounds[1][1], coord[1]);
                            });
                        });
                    } else if (feature.geometry.type === "MultiPolygon") {
                        feature.geometry.coordinates.forEach((coords) => {
                            coords.forEach((coords) => {
                                coords.forEach((coord) => {
                                    bounds[0][0] = Math.min(
                                        bounds[0][0],
                                        coord[0]
                                    );
                                    bounds[0][1] = Math.min(
                                        bounds[0][1],
                                        coord[1]
                                    );
                                    bounds[1][0] = Math.max(
                                        bounds[1][0],
                                        coord[0]
                                    );
                                    bounds[1][1] = Math.max(
                                        bounds[1][1],
                                        coord[1]
                                    );
                                });
                            });
                        });
                    }
                });
                // 返回的是左下，右上
                return bounds;
            }
  1. 仿照fill-pattern-repeat新增一个属性fill-bound 类型为Array,数据驱动，然后将计算好的东西传进去。有一点需要注意，这里我选择将经纬度转为墨卡托坐标，传了进去，严格意义讲这并不是Cesium那样的墨卡托坐标，它的单位是投影世界的宽度，不是墨卡托的米，是可直接用于webgl计算的坐标

由于流程类似，这里减少重复描述

计算瓦片范围 ​

只有图层范围无法计算图片UV，需要获取瓦片范围。

  1. 在Tile中，我们可以获取瓦片的行列号，通过计算，可以把这个行列号转换为墨卡托坐标（webgl）以下是计算方式
js
// pattern.js
// patternUniformValues
    const z2 = 1 << tile.tileID.canonical.z;
    // 左上 lt
    const lng = (tile.tileID.canonical.x) / z2;
    const lat = (tile.tileID.canonical.y) / z2;
    // 右下 rb
    const lng2 = (tile.tileID.canonical.x + 1) / z2;
    const lat2 = (tile.tileID.canonical.y + 1) / z2;
  1. 你可以选择将图层范围和瓦片范围分别传进去，也可以提前计算好传进去,这里选择后者
js
// pattern.js
// patternUniformValues
    return {
        'u_bound_lt': [(lng - bound[0]) / (bound[2] - bound[0]), (lat - bound[1]) / (bound[1] - bound[3])],
        'u_bound_rb': [(lng2 - bound[0]) / (bound[2] - bound[0]), (lat2 - bound[1]) / (bound[1] - bound[3])]
    };
  1. 顶点着色器中，计算uv坐标
js
// fill_pattern.vertex.glsl
uniform vec2 u_bound_lt;
uniform vec2 u_bound_rb;

varying vec2 v_uv;
// ....
v_uv = a_pos/8192.0 * (u_bound_rb - u_bound_lt) + u_bound_lt ;
  1. 片元着色器中，使用该uv替换原来使用的
js
// fill_pattern.fragment.glsl
  varying vec2 v_uv;
// 根据u_repeat执行不同代码，以保证旧有功能仍然有效
  vec2 imagecoord;
  if(!u_repeat){
      imagecoord  = mod(v_uv, 1.0);
  } else {
      imagecoord  = mod(v_pos, 1.0);
  }
  1. 你会发现v是反的，所以再取一次反即可。
js
// fill_pattern.fragment.glsl
  if(!u_repeat){
      pos.y = 1.0 - pos.y;
  }

重新调用，查看效果 ​

js
  map.addSource("china", {
      type: "geojson",

总结 ​

该功能能满足部分应用场景，也确实存在一些不便利性，不过对于学习mapbox底层逻辑是不错的入门。
